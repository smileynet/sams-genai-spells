# Documentation

## Tutorials — The concepts behind the spells

Each spell applies an established concept from a real field. These tutorials explain the concept itself, where you already encounter it, and how the spell applies it.

| Tutorial | The concept |
|----------|-------------|
| [Idiomatic Code](tutorials/idiomatic.md) | Every language has canonical patterns — style guides, linters, and conventions that go beyond syntax |
| [The Socratic Method](tutorials/socratic.md) | Teaching through guided questions instead of answers — used in law schools, coaching, and pair programming |
| [Best Practices & Antipatterns](tutorials/bpap.md) | The do/don't format plus named antipatterns — engineering standards, security checklists, and trap analysis |
| [Progressive Disclosure](tutorials/progressive-disclosure.md) | Simple first, complexity on demand — the UX principle behind doc-audit, doc-restructure, and doc-generate |
| [Diataxis](tutorials/diataxis.md) | Four-quadrant documentation (tutorials, how-to, reference, explanation) — the documentation theory behind doc-audit, doc-restructure, and doc-generate |
| [Task Graphs](tutorials/task-graph.md) | DAGs, topological sort, and critical paths — the same concepts behind build systems and package managers |
| [Systematic Diagnosis](tutorials/diagnose.md) | Root cause analysis using fishbone categorization, 5 Whys, and hypothesis-driven debugging |
| [Categorical Cause Decomposition](tutorials/cause-map.md) | Fishbone (Ishikawa) analysis — mapping all possible causes across categories before investigating |
| [Codebase Exploration](tutorials/deep-dive.md) | Program comprehension — systematic reading strategies for understanding unfamiliar code |
| [Prior Art Search](tutorials/prior-art.md) | Survey existing solutions before building — libraries, tools, frameworks, and patterns |
| [Blind Spot Detection](tutorials/blind-spot.md) | Pre-mortem analysis, assumption surfacing, and cross-domain transfer — finding what you don't know you don't know |
| [Problem Framing](tutorials/zoom-out.md) | Strategic thinking — reframing, inversion, opportunity cost, second-order effects, and upstream analysis |
| [Shift Handoff Protocols](tutorials/handoff.md) | Structured context transfer from medicine, aviation, and military — adapted for software and AI sessions |
